1. Toast Notification
Notifikasi kecil muncul di sudut layar dengan progress bar countdown. Hover untuk pause.
Tipe Toast
4 tipe dasar — success, danger, warning, info
Pilih Posisi:
// Toast sederhana DKA.toast({ type : 'success', // success | danger | warning | info title : 'Berhasil!', message : 'Data berhasil disimpan.', duration : 4000, // ms position : 'top-right', // top-right|top-left|top-center|bottom-* });
2. Dialog Konfirmasi
Dialog modal dengan Promise — tunggu jawaban user sebelum lanjut eksekusi.
Standard Dialog
Semua tipe dialog — hasilnya tampil di console bawah
// Hasil dialog akan tampil di sini...
DKA.dialog({ type : 'danger', title : 'Konfirmasi Aksi', message: 'Apakah Anda yakin ingin melanjutkan?', confirm: 'Ya, Lanjutkan', cancel : 'Batal', }).then(result => { if (result) { // user klik konfirm } });
3. Delete Confirm
Dialog hapus khusus dengan chip nama item yang akan dihapus — lebih informatif dan aman.
Alert Delete
Konfirmasi penghapusan dengan nama item yang ditampilkan
// Hasil delete confirm akan tampil di sini...
DKA.deleteConfirm({ title : 'Hapus Pengguna?', message : 'Tindakan ini tidak dapat dibatalkan.', itemName: 'Ahmad Fauzi — Administrator', confirm : 'Ya, Hapus Sekarang', cancel : 'Batal', }).then(result => { if (result) { // lanjutkan proses hapus } });
4. Approve User
Dialog persetujuan akun pengguna baru — menampilkan card info user lengkap sebelum dikonfirmasi.
Popup Approve Pengguna
Tampil kartu info pengguna sebelum disetujui
// Hasil approve akan tampil di sini...
DKA.approveUser({ title : 'Setujui Pengguna?', message: 'Pengguna akan mendapat akses ke sistem.', user: { name : 'Budi Santoso', role : 'Operator', email : 'budi@pekalongan.go.id', instansi : 'Dinas Sosial', avatarGrad: 'linear-gradient(135deg,#10b981,#06b6d4)', }, confirm: 'Ya, Setujui Akun', }).then(result => { if (result) { // akun disetujui → update status } });
5. Popup Ubah Password
Dialog ganti password lengkap — validasi real-time, strength meter, rules checklist, toggle show/hide.
Password Dialog
Versi dengan password saat ini, dan versi tanpa (set password baru)
// Hasil password dialog akan tampil di sini...
DKA.password({ title : 'Ubah Password', confirm : 'Simpan Password Baru', cancel : 'Batal', requireCurrent: true, // false = tanpa password lama }).then(result => { if (result.confirmed) { // result.password berisi password baru console.log(result.password); } });
6. Loading Overlay
Overlay loading dengan 3 gaya animasi berbeda — ring, dots, wave. Update pesan secara dinamis.
Loading Overlay
3 style: ring spinner, dots pulse, wave bars
const loader = DKA.loading({ title : 'Menyimpan data...', message: 'Mohon tunggu sebentar.', style : 'ring', // 'ring' | 'dots' | 'wave' }); // Update pesan saat loading berlangsung setTimeout(() => loader.update('Memvalidasi data...'), 1500); setTimeout(() => loader.close(), 3000);
7. Notify Panel
Panel notifikasi besar yang muncul dari sisi kanan layar — cocok untuk informasi penting.
Notify (Side Panel)
Notifikasi besar dengan timestamp dan progress bar
DKA.notify({ type : 'success', title : 'Upload Berhasil', message : '8 foto kegiatan berhasil diupload.', duration: 6000, });
8. Inline Alert
Alert yang diinjeksi langsung ke dalam elemen target di halaman — tidak muncul sebagai overlay.
Inline Alert
Alert muncul di dalam area konten di bawah ini
Alert akan muncul di sini
DKA.alert({ type : 'success', title : 'Berhasil Disimpan', message : 'Data berhasil diperbarui.', closable: true, }, '#target-element'); // CSS selector atau element
9. Use Case Nyata
Contoh kombinasi DKA Alert dalam skenario aplikasi nyata DokaKegiatan.
Skenario Lengkap
Kombinasi loading → dialog → toast seperti di aplikasi sesungguhnya
Kontrol Global
Tutup semua elemen DKA yang sedang aktif
DKA.close(); // tutup dialog/loading aktif DKA.closeAll(); // tutup semua: backdrop + toast + notify
API Reference
Dokumentasi lengkap semua method yang tersedia di DKA library.
Method Parameter Utama Return Keterangan
DKA.toast() type, title, message, duration, position { dismiss } Toast notifikasi kecil di sudut layar
DKA.dialog() type, title, message, confirm, cancel Promise<boolean> Dialog konfirmasi standar dengan 5 tipe
DKA.deleteConfirm() title, message, itemName, confirm, cancel Promise<boolean> Dialog hapus khusus dengan chip nama item
DKA.approveUser() title, message, user{name,role,email,instansi,avatarGrad} Promise<boolean> Dialog approve pengguna dengan kartu info
DKA.password() title, confirm, cancel, requireCurrent Promise<{confirmed,password}> Dialog ubah password dengan strength meter & validasi
DKA.loading() title, message, style ('ring'|'dots'|'wave') { close(), update(msg) } Overlay loading, update pesan secara dinamis
DKA.notify() type, title, message, duration { dismiss } Panel notifikasi besar dari sisi kanan
DKA.alert() type, title, message, closable + targetEl { el, dismiss() } Alert inline diinjeksi ke elemen target
DKA.close() — void Tutup backdrop aktif (dialog/loading)
DKA.closeAll() — void Tutup semua elemen DKA yang aktif
